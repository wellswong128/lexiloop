# LexiLand Data Model

This document describes the data model for the first localStorage version and the future Supabase version.

The main idea is simple: start with one `Word` object stored in localStorage, then later map the same fields to database tables.

## Core Entity: Word

A word represents one vocabulary item saved by the user.

```js
const word = {
  id: "word_1710000000000_abc123",
  term: "resilient",
  definition: "Able to recover quickly from difficulty.",
  translation: "有韌性的",
  pronunciation: "/rɪˈzɪliənt/",
  partOfSpeech: "adjective",
  example: "She is resilient after every setback.",
  notes: "Commonly used to describe people, systems, or communities.",
  tags: ["personality", "advanced"],
  source: "manual",
  createdAt: "2026-06-09T09:00:00.000Z",
  updatedAt: "2026-06-09T09:00:00.000Z",
  review: {
    level: 0,
    nextReviewAt: "2026-06-09T09:00:00.000Z",
    lastReviewedAt: null,
    correctCount: 0,
    incorrectCount: 0,
    lastResult: null
  },
  mistake: {
    isMistake: false,
    lastMistakeAt: null,
    mistakeCount: 0
  }
};
```

## Word Fields

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | string | Yes | Unique ID generated by the app. |
| `term` | string | Yes | The English word or phrase. |
| `definition` | string | Yes | English definition. |
| `translation` | string | No | Chinese translation or learner's native language. |
| `pronunciation` | string | No | IPA or simple pronunciation note. |
| `partOfSpeech` | string | No | Example: noun, verb, adjective. |
| `example` | string | No | Example sentence. |
| `notes` | string | No | User notes. |
| `tags` | string[] | No | Learning labels such as topic or level. |
| `source` | string | Yes | `manual`, `import`, or later `ai`. |
| `createdAt` | ISO string | Yes | Creation time. |
| `updatedAt` | ISO string | Yes | Last update time. |
| `review` | object | Yes | Spaced repetition state. |
| `mistake` | object | Yes | Mistake notebook state. |

## Review Fields

| Field | Type | Meaning |
| --- | --- | --- |
| `level` | number | Current spaced repetition level. Starts at `0`. |
| `nextReviewAt` | ISO string | The next date when this word should be reviewed. |
| `lastReviewedAt` | ISO string or null | Last review time. |
| `correctCount` | number | Total correct answers. |
| `incorrectCount` | number | Total incorrect answers. |
| `lastResult` | string or null | `correct`, `incorrect`, `remembered`, or `forgot`. |

## Mistake Fields

| Field | Type | Meaning |
| --- | --- | --- |
| `isMistake` | boolean | Whether the word appears in the mistake notebook. |
| `lastMistakeAt` | ISO string or null | Last time the user got this word wrong. |
| `mistakeCount` | number | Total number of mistakes for this word. |

## Simple Spaced Repetition Rules

The first version should use a simple interval table.

| Level | Next Review |
| --- | --- |
| 0 | Today |
| 1 | Tomorrow |
| 2 | 3 days later |
| 3 | 7 days later |
| 4 | 14 days later |
| 5+ | 30 days later |

When the user remembers a word:

1. Increase `review.level` by 1.
2. Increase `review.correctCount` by 1.
3. Set `review.lastResult` to `remembered` or `correct`.
4. Set `review.lastReviewedAt` to now.
5. Set `review.nextReviewAt` based on the new level.

When the user forgets a word:

1. Set `review.level` back to `0` or reduce it by 1.
2. Increase `review.incorrectCount` by 1.
3. Set `review.lastResult` to `forgot` or `incorrect`.
4. Set `review.lastReviewedAt` to now.
5. Set `review.nextReviewAt` to tomorrow or today.
6. Set `mistake.isMistake` to `true`.
7. Increase `mistake.mistakeCount` by 1.
8. Set `mistake.lastMistakeAt` to now.

Recommended beginner rule: if forgotten, reset `level` to `0` and review again tomorrow.

## localStorage Design

Use one storage key for the MVP:

```text
lexiland.words.v1
```

The value should be a JSON string containing an array of `Word` objects.

```json
[
  {
    "id": "word_1710000000000_abc123",
    "term": "resilient",
    "definition": "Able to recover quickly from difficulty.",
    "translation": "有韌性的",
    "tags": ["advanced"],
    "source": "manual",
    "createdAt": "2026-06-09T09:00:00.000Z",
    "updatedAt": "2026-06-09T09:00:00.000Z",
    "review": {
      "level": 0,
      "nextReviewAt": "2026-06-09T09:00:00.000Z",
      "lastReviewedAt": null,
      "correctCount": 0,
      "incorrectCount": 0,
      "lastResult": null
    },
    "mistake": {
      "isMistake": false,
      "lastMistakeAt": null,
      "mistakeCount": 0
    }
  }

```

## Import JSON Format

Imported words can be simpler than internal app data. The app should fill missing system fields automatically.

```json
[
  {
    "term": "resilient",
    "definition": "Able to recover quickly from difficulty.",
    "translation": "有韌性的",
    "pronunciation": "/rɪˈzɪliənt/",
    "partOfSpeech": "adjective",
    "example": "She is resilient after every setback.",
    "notes": "Useful for describing people.",
    "tags": ["advanced", "personality"]
  }
]
```

Required import fields:

- `term`
- `definition`

Optional import fields:

- `translation`
- `pronunciation`
- `partOfSpeech`
- `example`
- `notes`
- `tags`

## Duplicate Word Rule

For the first version, duplicate detection can be simple:

1. Convert `term` to lowercase.
2. Trim extra spaces.
3. If another word has the same normalized term, treat it as duplicate.

Recommended MVP behavior: skip duplicates during import and show how many were skipped.

## Future Supabase Tables

The localStorage model can later be split into relational tables.

### `profiles`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | uuid | Same as Supabase auth user ID. |
| `display_name` | text | Optional. |
| `created_at` | timestamptz | Default now. |
| `updated_at` | timestamptz | Updated when profile changes. |

### `words`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | uuid | Primary key. |
| `user_id` | uuid | References auth user. |
| `term` | text | Required. |
| `definition` | text | Required. |
| `translation` | text | Optional. |
| `pronunciation` | text | Optional. |
| `part_of_speech` | text | Optional. |
| `example` | text | Optional. |
| `notes` | text | Optional. |
| `tags` | text[] | Optional. |
| `source` | text | `manual`, `import`, or `ai`. |
| `review_level` | integer | Default `0`. |
| `next_review_at` | timestamptz | Used to find due words. |
| `last_reviewed_at` | timestamptz | Optional. |
| `correct_count` | integer | Default `0`. |
| `incorrect_count` | integer | Default `0`. |
| `is_mistake` | boolean | Default `false`. |
| `last_mistake_at` | timestamptz | Optional. |
| `mistake_count` | integer | Default `0`. |
| `created_at` | timestamptz | Default now. |
| `updated_at` | timestamptz | Updated when word changes. |

### `review_events`

Use this later if you want review history and charts.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | uuid | Primary key. |
| `user_id` | uuid | References auth user. |
| `word_id` | uuid | References `words.id`. |
| `result` | text | `correct`, `incorrect`, `remembered`, or `forgot`. |
| `review_mode` | text | `flashcard` or `quiz`. |
| `created_at` | timestamptz | Time of review. |

## Supabase Row Level Security

When Supabase is added, every user should only access their own data.

Example rule idea:

- Users can select their own words where `user_id = auth.uid()`.
- Users can insert words where `user_id = auth.uid()`.
- Users can update their own words where `user_id = auth.uid()`.
- Users can delete their own words where `user_id = auth.uid()`.

## AI Auto-Completion Data

Future AI output should match the import format, not the full internal model.

Expected AI suggestion:

```json
{
  "term": "resilient",
  "definition": "Able to recover quickly from difficulty.",
  "translation": "有韌性的",
  "pronunciation": "/rɪˈzɪliənt/",
  "partOfSpeech": "adjective",
  "example": "She is resilient after every setback.",
  "tags": ["advanced", "personality"]
}
```

The app should show AI output as editable form values. The app should not save AI output automatically without user confirmation.
